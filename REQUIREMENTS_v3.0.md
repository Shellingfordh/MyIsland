# REQUIREMENTS v3.0 (Context Hub, 引流版最终形态)

> 目标：这是进入引流阶段前的“最终大版本”。功能做全、体验做稳、安全边界做清楚。此后不再新增大功能，仅做维护与修复。

## 产品定位
MyIsland v3.0 是 **macOS 菜单栏“上下文 + 热度”中枢**：
- 既像原生 Siri 药丸，又是轻量的“系统状态 + 热点摘要”入口。
- 强调 **只读信息、低风险、强可控**，不做隐式自动化。

## 核心目标
1. **上下文感知**：当前 App、媒体、专注、系统状态一目了然。
2. **热度摘要**：GitHub Trending + Google Trends 的“可选”即时热度提示。
3. **性能稳定**：事件驱动为主、轮询为辅，避免常驻高频刷新。
4. **安全可控**：默认只读，任何具有动作的能力必须“明确确认”。

## 功能范围（最终版）

### 1) Context Cards（上下文卡片）
- **内容**：
  - 当前前台 App / 窗口标题（含浏览器站点识别）
  - 当前媒体（Spotify / Apple Music / 网易云等）
  - 系统专注模式（Focus）
  - 关键系统指标（电量、音量、亮度）
- **表现**：
  - 右键菜单内以 2~4 张卡片显示
  - 与现有胶囊主状态保持一致性
- **刷新策略**：事件触发优先；周期性刷新仅用于兜底

### 2) Hotness Tray（热度托盘）
- **数据源**：
  - GitHub Trending（Top N 项目）
  - Google Trends Trending Now（Top N 关键词）
- **模式**：
  - `off`（默认）
  - `on`（开启）
- **缓存策略**：
  - 15~30 分钟本地缓存（可配置）
  - 网络失败回退到最近缓存
- **可视化**：
  - 右键菜单“热度托盘”面板
  - 每条仅显示标题 + 简短补充（语言/热度标记）

### 3) Agent 只读模式（Final）
- 仅输出 **上下文摘要** 或 **热度摘要**
- 不允许触发系统级动作或自动化脚本
- 用户主动点击后才可复制/查看详情

### 4) Drop Box & Control Center（保留并完善）
- Drop Box 作为安全的“文件投递箱”保留
- 右键菜单继续提供：
  - 快捷入口（Drop Box / Flux / Quick Switch / Glass）
  - 安全与隐私提示（只读模式、无隐式自动化）

## 体验与交互规范
1. **只读优先**：所有信息展示默认只读
2. **动作确认**：任何执行动作的入口必须显示确认对话框
3. **低干扰**：胶囊主状态不频繁闪动，动画仅限边框
4. **可关闭**：Hotness 一键关闭，Context 可降级展示

## 配置与扩展
新增 `config.json`（或扩展 `userconfig.sh`）配置项：
- `HOTNESS_ENABLED`：on/off
- `HOTNESS_CACHE_MINUTES`：15/30/60
- `HOTNESS_SOURCES`：github, google
- `CONTEXT_CARDS`：app/media/focus/system
- `UI_LANGUAGE`：auto/zh/en

## 性能与安全
- 热度请求均通过本地脚本拉取并缓存
- 网络失败不得阻塞 UI
- 不读取或上传隐私数据
- 不执行隐式自动化

## 兼容性
- macOS 12+（建议 13+）
- Apple Silicon / Intel
- Notch / Non‑Notch
- SketchyBar 最新稳定版

## 验收标准
1. Hotness 关闭时：无任何网络请求；UI 不出现空白/闪烁
2. Hotness 开启时：30 秒内展示内容；失败时展示缓存
3. Context Cards：对 App / Music / Focus 变化响应 < 1 秒
4. 性能：CPU 占用稳定，长期运行不导致卡顿
5. 安全：无自动执行行为，所有动作需明确确认

## 非目标（明确不做）
- 不做自动化执行（包括自动点击/自动控制系统）
- 不做隐式账号登录或验证码处理
- 不做后台常驻高频抓取
- 不做大而全的“聊天机器人”界面

## 里程碑建议（已锁定范围）
1. **v3.0-alpha**：Hotness Tray + 缓存 + UI 框架
2. **v3.0-beta**：Context Cards + 事件优化
3. **v3.0**：引导、配置、最终稳定与文档完善
